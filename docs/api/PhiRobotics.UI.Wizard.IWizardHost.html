Provides functionality to execute wizards discovered from wizard assemblies.

The IWizardHost interface is implemented by the UI library and is not intended to be implemented by consumer code.

Remarks

The IWizardHost is responsible for locating and executing wizard implementations that are registered with the host.

Implementations typically load wizard assemblies from a configured directory and create wizard instances when requested.

Wizards are created and executed in a separate AppDomain to isolate them from the main application. As a result, wizard assemblies are loaded into the isolated domain rather than the primary application domain.

Running wizards in an isolated application domain improves reliability and stability. If a wizard throws an unhandled exception or fails during execution, the failure is contained within the isolated domain and does not crash the main application.

This isolation also allows wizard assemblies to be unloaded when the IWizardHost instance is disposed.

Consumer code obtains an instance of IWizardHost using the CreateHost(string, ILoggingService) method. The lifetime of the host instance is managed by the consumer.

Show(Guid, string, Image, object)

Runs the wizard identified by the specified wizard identifier.

WizardResult Show(Guid wizardId, string dialogCaption = null, Image dialogIcon = null, object initArgs = null)

Parameters

wizardId Guid

The unique identifier of the wizard to execute. This identifier must match the identifier of a wizard registered with the wizard host.

dialogCaption string

Optional caption text displayed in the wizard dialog title bar. If null or empty, a default caption may be used.

dialogIcon Image

Optional icon displayed in the wizard dialog. If null, a default icon may be used.

initArgs object

Optional initialization data passed to the wizard through Initialize(object) before the wizard is displayed.

Returns

WizardResult

A WizardResult containing the dialog result and any data produced by the wizard.

Remarks

This method displays the wizard dialog to the user, manages navigation between wizard pages, and handles user interactions such as Next, Back, Finish, and Cancel.

If the wizard host is unable to create an IWizard instance using the registered wizard information, if the Pages property returns no pages, or if any method or property of the IWizard throws an exception during execution, the wizard dialog is not displayed. In such cases, the wizard is marked as having an error and will not be executed again during the lifetime of the host.

The returned WizardResult indicates how the wizard dialog was closed. When the wizard completes successfully (OK), the Data property may contain the data produced by the wizard. For other dialog results, this value is null.

Exceptions

ArgumentException

Thrown if wizardId is Empty, or does not correspond to a registered wizard.

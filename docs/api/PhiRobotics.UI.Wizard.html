Namespace PhiRobotics.UI.Wizard

Provides infrastructure for discovering, loading, and executing wizards within the PhiRobotics UI library.

The wizard enables applications to extend functionality through modular wizard plugins. A wizard guides users through a sequence of steps using a structured user interface consisting of a welcome page, one or more intermediate pages, and a completion page.

Wizards are implemented by consumer code through the IWizard interface and are packaged in external assemblies. These assemblies are discovered and loaded at runtime.

Each wizard must be exported using the ExportedWizardAttribute assembly attribute and identified by a unique identifier using the WizardIdAttribute. During startup, the wizard host scans configured directories for assemblies containing exported wizards and registers them for execution.

Wizards are executed through the IWizardHost service, which is created by IWizardFactory. The wizard host is responsible for locating wizard implementations, creating wizard instances, and displaying the wizard dialog to the user.

To improve reliability and stability, wizard implementations are executed in an isolated AppDomain. This isolation ensures that wizard assemblies are not loaded into the main application domain and that failures inside a wizard do not crash the main application. When the wizard host is disposed, the isolated domain can be unloaded, allowing all wizard assemblies to be released.

The wizard user interface is presented using IWizardDialog, which manages navigation between pages and handles user interactions such as Next, Back, Finish, and Cancel.

The result of a wizard execution is returned as a WizardResult, which contains the dialog result and any data produced by the wizard.

Classes

CompletionPage

Represents the completion page of a wizard, typically displayed as the final step.

ExportedWizardAttribute

Indicates that a wizard type is exported from an assembly.

WelcomePage

Represents the welcome page of a wizard, typically displayed as the first step.

WizardIdAttribute

Specifies the identifier of a wizard class.

WizardResult

Represents the result returned after executing a wizard.

Interfaces

IWizard

Represents a wizard and provides access to its pages and lifecycle methods.

The IWizard interface is intended to be implemented by consumer code. The lifetime of the wizard instance is managed by the consumer.

IWizardDialog

Represents a dialog used to display and execute a wizard.

The IWizardDialog interface is implemented by the PhiRobotics UI library and is not intended to be implemented by consumer code.

IWizardFactory

Factory that provides methods to create instances of IWizardHost used to discover and execute wizards, and to create instances of IWizardDialog used to display wizard dialogs.

The IWizardFactory interface is implemented by the PhiRobotics UI library and is not intended to be implemented by consumer code.

IWizardHost

Provides functionality to execute wizards discovered from wizard assemblies.

The IWizardHost interface is implemented by the UI library and is not intended to be implemented by consumer code.

IWizardPage

Represents an interior page in a wizard, typically displayed between the WelcomePage and CompletionPage.

The IWizardPage interface is intended to be implemented by consumer code. The lifetime of the page instance is managed by the consumer.

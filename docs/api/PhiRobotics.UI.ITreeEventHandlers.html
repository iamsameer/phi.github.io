Interface ITreeEventHandlers

Defines handlers for tree-related events raised by an ITree.

Namespace: PhiRobotics.UI
Assembly: PhiRobotics.UI.dll
Syntax
public interface ITreeEventHandlers
Remarks

Implement this interface to receive notifications about user interactions and state changes in the tree, such as node selection changes, expansion, editing, and drag-and-drop operations.

OnBeforeExpandNode(ITree, BeforeExpandNodeEventArgs)

Called before a node is expanded.

Declaration
void OnBeforeExpandNode(ITree tree, BeforeExpandNodeEventArgs args)
Parameters
Type Name Description
ITree tree

The tree that raised the event.
BeforeExpandNodeEventArgs args

Provides information about the node being expanded.
Remarks

This method allows you to implement dynamic (lazy) loading of child nodes.

The method is called even if the node does not currently contain child nodes, provided that its HasChildren property is set to true.

A typical dynamic loading pattern:

  1. Create only root nodes initially.
  2. Set HasChildren to true.
  3. Load child nodes inside the OnBeforeExpandNode handler.

Remarks

The OnDragDrop method is called when a drag-and-drop operation completes inside the tree.

The drop location and target node are provided through TreeHitTestData.

You can evaluate keyboard modifiers (SHIFT, CTRL, ALT) to determine the intended operation and set DropEffect accordingly.

Declaration
void OnDragOver(ITree tree, TreeDragOverEventArgs args)
Parameters
Type Name Description
ITree tree

The tree that raised the event.
TreeDragOverEventArgs args

Provides information about the current drag location and allows the handler to control whether dropping is allowed.
Remarks

The OnDragOver method is called when the mouse cursor moves within the bounds of the tree during a drag-and-drop operation.

Consumers can evaluate the keyboard state to determine the intended drag operation based on the state of the SHIFT, CTRL, ALT, or CTRL+ALT keys.

The node and location where the drop would occur are determined using TreeHitTestData.

Consumers can set Effect to a value from DragDropEffects based on the keyboard state and intended action.

OnDragStarting(ITree, TreeDragStartedEventArgs)

Called when a drag operation starts from the tree.

Declaration
void OnDragStarting(ITree tree, TreeDragStartedEventArgs args)
Parameters
Type Name Description
ITree tree

The tree that raised the event.
TreeDragStartedEventArgs args

Provides information about the nodes being dragged and allows customization of the drag operation.
Remarks

The OnDragStarting method is called each time an attempt is made to start dragging one or more nodes.

You can use this method to set DataObject and AllowedEffects.

To prevent nodes from being dragged, set AllowedEffects to None.

The collection of nodes being dragged and the primary node can be accessed through Nodes and Node, respectively.

If you need to perform actions after the drag-and-drop operation completes, handle the OnDragCompleted(ITree, TreeDragCompletedEventArgs) method.

void OnFocusNodeChanged(ITree tree, FocusedNodeChangedEventArgs args)
Parameters
Type Name Description
ITree tree

The tree that raised the event.
FocusedNodeChangedEventArgs args

Provides information about the previous and new focused nodes.
Remarks

The OnFocusNodeChanged method is called when focus moves from one node to another for the following reasons:

  • A user moves focus to a node.
  • The FocusedNode property is changed programmatically.
  • Focus moves to the newly added node when the Append(string, object) method is used.
  • Focus moves to the parent of the focused node after the parent node is collapsed.
  • The currently focused node is removed from the tree.

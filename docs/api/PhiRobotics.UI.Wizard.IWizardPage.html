Represents an interior page in a wizard, typically displayed between the WelcomePage and CompletionPage.

The IWizardPage interface is intended to be implemented by consumer code. The lifetime of the page instance is managed by the consumer. 

public interface IWizardPage

Remarks

A wizard can display any number of interior pages to guide the user through a multi-step process. Each page provides its own user interface, validation logic, navigation rules, and optional help content.

Implementations of this interface are responsible for:

  • Providing the UI control displayed in the wizard.
  • Validating user input before navigation.
  • Controlling navigation behavior using GetNextPage() and GetPreviousPage().
  • Enabling or disabling navigation buttons through AllowNext and AllowBack.

The wizard monitors AllowBackChanged and AllowNextChanged events to update the state of the navigation buttons in the wizard dialog.

The Commit() method is called when the user attempts to move to the next page or finish the wizard. If validation fails, the method should return false and provide an error message through CurrentError.

The following example shows how to create a wizard page: 
class RecipientDataPage : IWizardPage
{
    private bool allowBack = true;
    private bool allowNext = true;
    private RecipientDataPageControl control = null;
    private BirthdayWish birthdayWish;

    public RecipientDataPage(BirthdayWish birthdayWish)
    {
        this.birthdayWish = birthdayWish;
        control = new RecipientDataPageControl(this);
        control.CurrentErrorKind = ErrorKind.None;
        control.CurrentError = string.Empty;
    }

    #region IWizardPage
    public event EventHandler AllowBackChanged;
    public event EventHandler AllowNextChanged;

    public bool AllowBack
    {
        get { return allowBack; }
        set
        {
            if (allowBack != value)
            {
                allowBack = value;
                AllowBackChanged?.Invoke(this, EventArgs.Empty);
            }
        }
    }
    public bool AllowNext
    {
        get { return allowNext; }
        set
        {
            if (allowNext != value)
            {
                allowNext = value;
                AllowNextChanged?.Invoke(this, EventArgs.Empty);
            }
        }
    }
    public string HeaderText
    {
        get { return "Recipient"; }
    }
    public ErrorKind CurrentErrorKind
    {
        get { return control.CurrentErrorKind; }
    }
    public string CurrentError
    {
        get { return control.CurrentError; }
    }
    public string DescriptionText
    {
        get { return "Recipient Name and Address"; }
    }
    public Control Control
    {
        get { return control; }
    }
    public bool HasHelp
    {
        get { return true; }
    }

    public void Activating()
    {
        control.RecipientName = birthdayWish.RecipientName;
        control.EmailAddress = birthdayWish.RecipientEmail;
    }
    public bool Commit()
    {
        if (string.IsNullOrWhiteSpace(control.RecipientName))
        {
            return false;
        }
        else if (string.IsNullOrWhiteSpace(control.EmailAddress))
        {
            return false;
        }
        birthdayWish.RecipientName = control.RecipientName;
        birthdayWish.RecipientEmail = control.EmailAddress;
        return true;
    }
    public string GetNextPage()
    {
        return "CardTemplatePage";
    }
    public string GetPreviousPage()
    {
        return null;
    }
    public void ShowHelp()
    {
        MessageBox.Show("This is recipient page of wizard.", "Wizard Page",
            MessageBoxButtons.OK, MessageBoxIcon.Exclamation);
    }
    #endregion
}

CurrentError

Gets the current error message for the page, if any.

string CurrentError { get; }

Property Value

string

A string describing the current validation error on the page. Returns an empty string if no error exists.

Remarks

This property typically provides feedback to the user when the data entered on the page is invalid or incomplete. It is often displayed in the wizard dialog to guide the user in correcting the issue before proceeding.

The value is typically updated during Commit() when validation fails. Before validation occurs, this property may return an empty string or an invalid error message.

CurrentErrorKind

Gets the kind or severity of the current error for the page.

ErrorKind CurrentErrorKind { get; }

Property Value

ErrorKind

An ErrorKind value that indicates whether the current state represents an informational message, warning, error, or no issue at all.

Remarks

This property typically works in conjunction with CurrentError to provide additional context about the nature of the problem or message.

If CurrentError is an empty string, this property will typically return None.

The description text helps users understand the purpose of the page or what information is required before proceeding to the next step.

Methods

Activating()

Called when the wizard is about to activate this page.

void Activating()

Remarks

This method is called by the wizard control immediately before the page becomes active. Use this method to initialize or refresh UI state, load data into the page controls, or reset validation state before the page becomes visible.

Commit()

Validates user input and persists any changes made on the page.

bool Commit()

Returns

bool

true if changes are successfully committed and the page is valid; otherwise, false.

Remarks

This method is called when the user presses the Next or Finish button. Implement this method to save user input, perform validation, or execute any finalization logic required before moving to the next page.

If validation fails or changes cannot be committed, return false and set the CurrentError property with an appropriate error message. This message will typically be displayed to the user in the wizard dialog.

GetNextPage()

Returns the next wizard page to navigate to.

string GetNextPage()

Returns

string

The name of the next page to display. This must match the key of a page in the Pages dictionary.

Remarks

The returned value must match the key of a page in the wizard's Pages collection.

Returning null or an empty string indicates that the wizard should navigate to the completion page.

If a non-empty value is returned, it must correspond to an existing page key in Pages; otherwise, navigation may fail.

Exceptions

InvalidOperationException

Thrown if AllowNext returns false.

GetPreviousPage()

Returns the previous wizard page to navigate to.

string GetPreviousPage()

Returns

string

The name of the previous page to display. This must match the key of a page in the Pages dictionary.

Remarks

The returned value must match the key of a page in the wizard's Pages collection.

Returning null or an empty string indicates that the wizard should navigate to the WelcomePage.

If a non-empty value is returned, it must correspond to an existing page key in Pages; otherwise, navigation may fail.

Exceptions

InvalidOperationException

Thrown if AllowBack returns false.

ShowHelp()

Displays help information related to this page.

void ShowHelp()

Remarks

This method is called when the user requests help (e.g., by clicking a Help button). The behavior of this method depends on the specific page implementation. It may open a help URL in a browser, display an additional dialog, or present context-sensitive help content. The wizard only calls this method if HasHelp returns true.

Exceptions

InvalidOperationException

Thrown if this method is called when HasHelp returns false.

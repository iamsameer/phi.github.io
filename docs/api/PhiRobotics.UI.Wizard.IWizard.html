Represents a wizard and provides access to its pages and lifecycle methods.

The IWizard interface is intended to be implemented by consumer code. The lifetime of the wizard instance is managed by the consumer. 

public interface IWizard

Remarks

The IWizard interface defines the structure and behavior of a wizard process, including its welcome page, completion page, and intermediate pages. It also provides lifecycle methods that allow the wizard to be initialized, cancelled, or completed.

A wizard consists of a sequence of pages that guide the user through a task step-by-step. The wizard host is responsible for displaying these pages and managing navigation between them.

Property Value

IReadOnlyDictionary<string, IWizardPage>

A read-only dictionary that maps page names (string) to their corresponding IWizardPage instances.

Remarks

This collection excludes the welcome and completion pages and typically contains the intermediate steps that guide the user through the wizard process.

This property should not be accessed before calling Initialize(object). Prior to initialization, the collection may be empty or contain incomplete page definitions.

Called whenever a wizard page becomes active in the wizard dialog. Implementations can use this notification to update shared state, validate data, or perform logic based on the currently displayed page.

Cancel()

void Cancel()

Remarks

Called by the IWizardHost when the user cancels or closes the wizard before completion.

Implementations should use this method to discard any changes made during the wizard process and release resources that were allocated during Initialize(object).

void Initialize(object data)

Parameters

data object

Remarks

This method is called by Show(Guid, string, Image, object) before displaying the wizard dialog. Implementations should use this method to prepare resources, initialize page data, or perform any setup logic required by the wizard.

The Pages property can be safely accessed only after initialization has completed.

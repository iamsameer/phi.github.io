Factory that provides methods to create instances of IWizardHost used to discover and execute wizards, and to create instances of IWizardDialog used to display wizard dialogs.

The IWizardFactory interface is implemented by the PhiRobotics UI library and is not intended to be implemented by consumer code. 

public interface IWizardFactory

Remarks

The IWizardFactory interface provides a mechanism to create wizard infrastructure components used by the application.

The created IWizardHost is responsible for discovering wizard assemblies, registering available wizards, and executing wizard dialogs.

Consumer code obtains an instance via the WizardFactory property. The lifetime of the factory instance is managed by the UI library.

Instances of IWizardHost and IWizardDialog created by this factory are owned by the consumer. The consumer is responsible for managing their lifetime and disposing them when they are no longer needed.

CreateHost(string, ILoggingService)

Creates a new IWizardHost instance.

IWizardHost CreateHost(string wizardsDirectory, ILoggingService loggingService = null)

Parameters

wizardsDirectory string

The directory containing wizard assemblies. The host scans this directory for assemblies that export wizard types using the ExportedWizardAttribute.

loggingService ILoggingService

An optional PhiRobotics.Services.ILoggingService used to record diagnostic information, such as wizard discovery results, loading errors, or runtime exceptions. If null, logging is disabled.

Returns

IWizardHost

An initialized IWizardHost that can be used to execute wizards discovered in the specified directory.

Remarks

During creation, the host scans the specified directory and registers all wizard types exported through the ExportedWizardAttribute assembly attribute.

Exceptions

ArgumentException

Thrown if wizardsDirectory is null, empty, or contains only whitespace characters.

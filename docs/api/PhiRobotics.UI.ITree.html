Represents a control that displays a hierarchical list of tree nodes.

The ITree interface is implemented by the UI library and is not intended to be implemented by consumer code.

Remarks

Instances of ITree are created using Create(ITreeEventHandlers).

The lifetime of an ITree instance is managed by the consumer. The consumer is responsible for disposing the tree when it is no longer needed.

The tree displays data using nodes. Use the Append(string, object) method to add a new node to the tree.

The RootNodes property provides access to nodes at the tree’s root level. Each node has an Nodes collection that stores its child nodes.

The following API can be used to retrieve nodes within the tree:

  • FocusedNode – Gets or sets the currently focused node.
  • NodeSelection – Gets the collection of selected nodes.

Property Value

Remarks

The ImageCollection property represents a storage used to resolve images based on the keys specified in ImageKey or ExpandedImageKey.

When a node's image needs to be displayed, the ITree control retrieves the image corresponding to the specified key from this collection.

This allows centralized management of images used by nodes in the tree.

Remarks

The RootNodes property returns a collection of root-level nodes. Each node in the collection may contain child nodes accessible through the Nodes property. Child nodes reference their parent through the ParentNode property.

Tree Traversal

You can enumerate nodes recursively by starting with a root node and then traversing its child nodes.

  • RootNodes – Provides access to root nodes.
  • Nodes – Provides access to a node’s child nodes. Use HasChildren to determine whether a node has children.
  • ParentNode – Provides access to a node’s parent.

Remarks

The BeginLoad and EndLoad methods internally call BeginUpdate() and EndUpdate(). Therefore, there is no need to call these methods explicitly when using BeginLoad and EndLoad.

BeginUpdate()

Locks the ITree, preventing visual updates of the object and its elements until the EndUpdate() or CancelUpdate() method is called.

void BeginUpdate()

Remarks

Each call to BeginUpdate must be paired with a call to EndUpdate() or CancelUpdate(). To ensure that EndUpdate/CancelUpdate is always called even if an exception occurs, use the try…finally statement.

To prevent excessive updates when the node structure is changed, use BeginLoad() and EndLoad()methods.

Parameters

point Point

The location to test, specified in client coordinates.

Returns

TreeHitTestData

A TreeHitTestData object that contains:

  • The Node associated with the specified point, if a node exists at that location.
  • The Location describing the exact part of the tree control at that position (for example, node content, expand button, or empty area).

Remarks

This method is typically used during mouse operations such as drag-and-drop to determine which node or tree element is located under a specific cursor position.

If the specified point does not correspond to a node, the returned Node may be null, while Location still describes the hit area.

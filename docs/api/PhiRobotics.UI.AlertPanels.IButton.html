Examples

The following example shows how to add custom buttons to an IAlertPanel. In the example, two custom buttons are added to the panel’s Buttons collection. The first button is a regular button, which when clicked, produces the ButtonClicked event. The second button has checked and unchecked states. Clicking this button fires the ButtonDownChanged event.

// Create alert panel.
IAlertPanel alertPanel = alertPanelFactory.CreatePanel();

// Create a regular custom button.
IButton button1 = alertPanelFactory.CreateButton("buttonOpen");
button1.Hint = "Open file";
button1.Image = Properties.Resources.Folder_16x16;

// Create a check custom button.
IButton button2 = alertPanelFactory.CreateButton("buttonAlert");
button2.IsCheckStyle = true;
button2.Down = true;
button2.Hint = "Alert On";
button2.Image = Properties.Resources.Timer_16x16;

// Add buttons to the AlertControl
alertPanel.Buttons.Add(button1);
alertPanel.Buttons.Add(button2);

// Subscribe to the events to process button clicks
alertPanel.ButtonClicked += AlertPanel_ButtonClicked;
alertPanel.ButtonDownChanged += AlertPanel_ButtonDownChanged;

// Show a sample alert window.
IAlertPanelInfo panelInfo = alertPanelFactory.CreatePanelInfo("New Window", "Text");
alertPanel.Show(this, panelInfo);

}

{

Remarks

Alert windows can display buttons that implement custom functionality. The IButton interface represents a custom button that can be shown within an alert window.

To add custom buttons, create IButton instances using CreateButton(string) and add them to the Buttons collection. Buttons in this collection are applied only to newly created alert windows. Modifying the collection does not affect buttons already displayed in existing alert windows.

An IButton can function either as a regular button or as a check (toggle) button. To enable check behavior, set the IsCheckStyle property to true. When enabled, the button maintains a pressed (Down) state.

You can customize button appearance and behavior using properties such as Name, Hint, Image, and ImageDown.

To handle user interaction, subscribe to the following events:

  • ButtonClicked — occurs when a regular button is clicked.
  • ButtonDownChanged — occurs when a check button’s state changes.

Properties

Down

Gets or sets whether the button is in the pressed (checked) state.

bool Down { get; set; }

Property Value

bool

Remarks

This property is effective only when IsCheckStyle is set to true. When enabled, the button behaves as a toggle (check) button and maintains its pressed state.

Hint

Gets or sets the tooltip text displayed for the button.

string Hint { get; set; }

Property Value

string

Remarks

Tooltips are displayed when the user hovers over the button. To disable tooltips, set the ShowToolTips property to false.

Image

Gets or sets the image displayed within the button.

Image Image { get; set; }

Property Value

Image

Remarks

If the button functions as a check button (IsCheckStyle is true), this image is also used for the pressed state unless a separate image is specified via the ImageDown property.

ImageDown

Gets or sets the image displayed when the button is in the pressed (checked) state.

Image ImageDown { get; set; }

Property Value

Image

Remarks

This property is applicable only when the button functions as a check button (IsCheckStyle is true).

If not specified, the image defined in the Image property is used for both normal and pressed states.

IsCheckStyle

Gets or sets whether the button behaves as a check (toggle) button.

bool IsCheckStyle { get; set; }

Property Value

bool

Remarks

When set to true, the button maintains a pressed state and toggles its Down property when clicked.

Name

Property Value

string

Remarks

The Name property is used to uniquely identify the button, especially when handling events such as ButtonClicked and ButtonDownChanged.

Predefined

Gets a value indicating whether this button is a predefined button.

bool Predefined { get; }

Property Value

bool

Remarks

Alert panels can display both predefined buttons (such as Close and Pin) and custom buttons. If this property is true, the button is one of the predefined buttons; otherwise, it is a custom button.

To control the visibility of predefined buttons, use the ShowCloseButton and ShowPinButton properties.

To add custom buttons, use the Buttons collection.

Represents different logical regions within an ITree that can be identified during a hit test operation.

The ITreeHitTestLocation interface is implemented by the UI library and is not intended to be implemented by consumer code. 

public interface ITreeHitTestLocation

Examples

The following example demonstrates how to use HitTest(Point) to determine which part of the tree is located under the mouse pointer.

TreeHitTestData hit = tree.HitTest(mousePoint);

if (hit.Location.IsText)
{
    Console.WriteLine($"Node text clicked: {hit.Node.Text}");
}

if (hit.Location.IsExpandButton)
{
    Console.WriteLine("Expand/collapse button clicked.");
}

if (hit.Location.IsFreeArea)
{
    Console.WriteLine("Clicked on empty space.");
}

Remarks

An ITreeHitTestLocation describes the specific region of the tree control located at the coordinates supplied to HitTest(Point).

A hit test location does not represent a node itself—it describes where within the tree the pointer is positioned (for example, over text, an image, an expand button, or an empty area).

This interface is typically used together with TreeHitTestData, which provides:

  • The ITreeNode (if any) at the specified position.
  • The ITreeHitTestLocation describing the exact region within the tree.

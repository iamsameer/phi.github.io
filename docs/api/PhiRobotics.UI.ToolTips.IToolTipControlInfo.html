Interface IToolTipControlInfo

Namespace
PhiRobotics.UI.ToolTips
Assembly

Provides tooltip information used by the tooltip controller to display a hint.

This interface is implemented by the PhiRobotics UI Library and should not be implemented by user code. 

public interface IToolTipControlInfo

Examples

The following example demonstrates how to display a tooltip dynamically for a RichTextBox based on the character under the mouse cursor.

private RichTextBox richTextBox1;
private IToolTipController controller;
private IToolTipFactory toolTipFactory;

private void StartTooltipForRichTextBox(IUIFactoryService uIFactoryService)
{
    toolTipFactory = uIFactoryService.ToolTipFactory;
    controller = toolTipFactory.CreateController();

    richTextBox1.MouseMove += richTextBox1_MouseMove;
}

private void richTextBox1_MouseMove(object sender, MouseEventArgs e)
{
    int chrIndex = richTextBox1.GetCharIndexFromPosition(e.Location);
    char chr = richTextBox1.GetCharFromPosition(e.Location);

    if (chrIndex < 0) { return; }

    string currentToolTipText = string.Format("Position: {0}, Character: {1}", chrIndex, chr);

    IToolTipControlInfo info = toolTipFactory.CreateControlInfo();
    info.Object = currentToolTipText;
    info.Text = currentToolTipText;
    info.ToolTipPosition = System.Windows.Forms.Form.MousePosition;

    controller.ShowHint(info);
}

Remarks

The Object property uniquely identifies the visual element for which the tooltip is displayed.

The Object property and at least one of Title or Text must be set; otherwise, the tooltip will not be displayed.

To create an instance of IToolTipControlInfo, use the CreateControlInfo() method.

ForcedShow

Gets or sets whether the tooltip should be forcibly shown for the same visual element via the ShowHint(IToolTipControlInfo) method.

ForcedShowKind ForcedShow { get; set; }

Property Value

ForcedShowKind

A ForcedShowKind enumeration value that specifies whether the tooptip should be forcibly shown for the same visual element.

Remarks

The Object property identifies the visual element for which the tooltip should be shown. If the ForcedShow property is set to Default or No, the IToolTipController shows the tooltip for this visual element only as result of the first call to the ShowHint(IToolTipControlInfo) method. Further calls to this method are not in effect until the active object is changed. Set the ForcedShow property to Yes to forcibly show the tooltip for the same visual element via the ShowHint(IToolTipControlInfo) method even if the tooltip has already been shown.

The ForcedShow property is not intended for tooltips supplied by QueryObjectInfo event handlers. These tooltips are shown when the mouse hovers over the required visual element. See the QueryObjectInfo event for details.

HideHintOnMouseMove

Gets or sets whether the shown tooltip hides when an end-user moves the mouse.

bool HideHintOnMouseMove { get; set; }

Property Value

bool

A bool value specifying whether the shown tooltip hides when an end-user moves the mouse.

Remarks

Use the HideHintOnMouseMove property to specify whether the displayed tooltip is hidden when an end-user moves the mouse. If this property is set to true, the tooltip hides immediately when an end-user starts moving the mouse. If it is set to false, the tooltip does not hide until the mouse pointer leaves the control for which the tooltip is shown.

To prevent a tooltip from being hidden on mouse moving, handle the QueryObjectInfo event, retrieve IToolTipControlInfo instance via Data's Info property and set its HideHintOnMouseMove property to false.

Interval

Gets or sets the interval that must pass before a tooltip is displayed.

int Interval { get; set; }

Property Value

int

An int value that specifies the delay, in milliseconds.

Remarks

By default, a tooltip is displayed onscreen after a delay which is determined by the ToolTipController’s InitialDelay and ReshowDelay settings.

The Interval property allows you to override these settings and provide a custom delay. If the Interval property is set to a positive value the delay will be determined by this property, not by the ToolTipController.

If the ShowImmediately property is set to true the Interval property has no effect.

Remarks

The value assigned to the Text property is used when the ToolTipType is set to Standard.

When the ToolTipType is set to RichTip and the RichToolTip property is null, the Title and Text properties are used by the tooltip controller to automatically construct and display a rich tooltip.

Ensure that at least one of the Title or Text properties is assigned a non-empty value; otherwise, the tooltip will not be displayed.
